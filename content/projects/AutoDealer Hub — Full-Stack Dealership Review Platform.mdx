## Overview

AutoDealer Hub is a distributed full-stack web application that connects customers with automobile dealerships through an interactive review platform. Users can browse dealerships, explore vehicle inventory, write reviews, and view AI-powered sentiment analysis of customer feedback.

Rather than building a traditional monolithic application, the project adopts a **microservices architecture**, separating authentication, dealership data, sentiment analysis, and frontend presentation into independent services.

---

## The Challenge

Modern web applications often require multiple independent services working together rather than a single backend.

The challenge was to answer:

> **How can multiple technologies be integrated into a scalable platform while keeping services modular and maintainable?**

The objective was to build an end-to-end dealership review system that demonstrates full-stack development, REST APIs, distributed services, authentication, and cloud deployment concepts.

---

## Design Approach

The system was designed around four principles:

- **Service Separation** — Authentication, dealership management, and sentiment analysis operate independently.
- **Scalability** — Each service can be deployed and scaled separately.
- **RESTful Communication** — Services communicate through lightweight HTTP APIs.
- **Containerized Deployment** — Docker and Kubernetes simplify deployment across environments.

## Technical Implementation

The project integrates multiple technologies into a cohesive distributed system.

Core implementation includes:

- React Single Page Application
- Django authentication and REST endpoints
- Node.js & Express dealership service
- MongoDB for dealership and review storage
- Flask sentiment-analysis microservice
- NLTK-based review classification
- SQLite for inventory and user models
- Docker containerization
- Kubernetes deployment configuration

---

## Engineering Challenges

### Multi-Service Communication

The application coordinates requests across Django, Node.js, Flask, and MongoDB while maintaining a seamless user experience.

### Authentication

Django manages user authentication and session handling while exposing secure endpoints for frontend interactions.

### Distributed Data Flow

Dealer information, customer reviews, and sentiment analysis originate from different services and are combined into a unified response before reaching the frontend.

### Containerized Deployment

Running multiple independent services required Docker-based environments and Kubernetes deployment manifests for orchestration.
